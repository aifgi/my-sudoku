## Learnings

- Project is a blank slate — no source files, only IntelliJ IDEA config; full stack freedom
- IntelliJ workspace.xml shows both TypeScript and Kotlin indices — developer likely comfortable with both
- Target: desktop only (Windows/Linux/macOS), single binary
- CRITICAL TENSION: Compose Multiplatform Desktop (best Kotlin GUI) requires JVM — produces a ~100-150MB installer, NOT a true single zero-dependency binary
- True single binary options for Kotlin: GraalVM native-image (Compose not supported) or TUI only
- True single binary with full GUI requires Go (Ebiten) or Rust (egui/iced) — different language from Kotlin
- ComposeArcade is the closest prior art: KMP Sudoku, MVI architecture, Android + Desktop, 99% shared code
- LibreSudoku is mature Android-only reference: 4 difficulties, stats, tutorials, custom puzzles
- Puzzle generation: Las Vegas fill + dig-holes + early-exit uniqueness check is the standard approach
- Difficulty grading: technique-based (naked singles → pairs → triples → X-Wing); no mathematical standard
- Hint system: constraint propagation solver run step-by-step, expose simplest applicable technique
- Dancing Links is fastest for uniqueness checks during generation; naive DFS is acceptable alternative
- No off-the-shelf KMP Sudoku engine library exists; must write from scratch

## Learnings (design phase — architect)
- Canvas-based board rendering chosen over LazyVerticalGrid: three simultaneous visual state layers (fill + border variation) and custom thick/thin grid lines require fine-grained draw control that Canvas provides cleanly
- IntArray(81) flat representation with bitmask candidates (int per cell, bits 1–9) avoids boxing overhead in hot conflict/hint paths; immutability is contractual not structural
- MRV (minimum remaining values) heuristic in backtracking solver eliminates need for Dancing Links — uniqueness checks empirically fast enough (<200ms); if Expert generation exceeds 2s, add forward-checking before considering DLX
- Difficulty-targeted generation uses retry loop (fill → grade → retry if mismatch) rather than targeted hole-digging — simpler implementation, converges quickly in practice (Easy/Medium: ~1 retry; Hard/Expert: ~5–20 retries)
- `HintResult.NoHintForDifficulty` is a first-class sealed variant — not null, not an error code — maps directly to the US-9 AC-9.1 message wording in the UI
- No `AndroidX ViewModel` on Compose Desktop — ViewModel lifetime is tied to `rememberCoroutineScope()` or a `LaunchedEffect`-anchored scope at the root composable
- Three cell visual states use: blue fill + solid border (selected), amber fill + crosshatch overlay (number-match), red fill + dashed border (conflict) — satisfies NFR-012 without color-only reliance
- WiX 4 confirmed needed on Windows CI runner for .msi — `choco install wixtoolset` or dedicated GitHub Action required
- `jpackage` module list (`--add-modules`) can reduce installer from ~250MB to ~100–150MB by stripping unused JDK modules
