## Learnings
- Root cause 1: explanation `Text` in `HintBanner.kt` (line 52) has no `weight(1f)` — both texts fight for width in the `Row`, causing the explanation to be squeezed/clipped
- Root cause 2: `Box` in `GameScreen.kt` line 173 uses `height(56.dp)` (exact/clipping) not `heightIn(min=56.dp)` (expandable)
- No `heightIn` precedent exists in codebase — all height usages are fixed `height(N.dp)`
- No UI/layout tests exist; bug is visual-only, verifiable only at runtime on device/emulator
- Longest hint strings are PointingPairCol EN (~95 chars) and RU (~96 chars) — easily 3 lines wide on a narrow right panel
- Russian Cyrillic glyphs are wider than Latin, making RU locale worse
- Screenshot confirms hint slot appears empty when a hint is active

## Requirements Learnings
- Scope is strictly 2-file, 2-line fix — intentionally no test requirement since no UI test infra exists
- `heightIn(min=56.dp)` has no prior usage in codebase but is the correct Compose idiom; assumption: no style guide prohibits it
- Assumption: technique name and explanation are already in separate `Text` composables in the same `Row`; `weight(1f)` on explanation is sufficient without restructuring
- RU locale is higher-risk than EN due to wider Cyrillic glyphs; both must be verified manually
